◆ add_recipe()

RecipesBook.main.views.add_recipe ( request) 
   Представлення для додавання нового рецепту.

   Це представлення дозволяє користувачеві додавати новий рецепт через форму. Якщо запит є POST-запитом,
   воно отримує дані з форми (назва, інгредієнти, інструкції, категорія, зображення) і створює новий запис у базі даних.
   Автором рецепту автоматично стає увійшовший користувач.

   Після успішного додавання рецепту користувач буде перенаправлений на сторінку "about-us".

   Якщо запит не є POST-запитом, буде відображена форма для додавання рецепту.

   Змінні контексту:
   - Немає змінних контексту, оскільки це форма для додавання нового рецепту.

   Аргументи:
       request (HttpRequest): Об'єкт запиту.

   Повертає:
       HttpResponseRedirect: Перенаправлення на сторінку "about-us" після успішного додавання рецепту.
       HttpResponse: Відображення форми для додавання рецепту, якщо запит не є POST-запитом.

◆ profile()

RecipesBook.main.views.profile ( request) 
    Представлення для відображення сторінки профілю користувача.

    Це представлення отримує та відображає інформацію, пов'язану з увійшовшим користувачем, включаючи:
    - Рецепти, створені користувачем.
    - Улюблені рецепти користувача.
    - Рецепти, які користувач хоче приготувати.
    - Список покупок на основі інгредієнтів рецептів, які користувач хоче приготувати.

    Список покупок формується шляхом ітерації по інгредієнтах рецептів, які користувач хоче приготувати,
    нормалізуючи назви інгредієнтів і підсумовуючи кількість для кожного інгредієнта.

    Форматований список покупок буде відображати кожен інгредієнт з його загальною кількістю, округленою до двох знаків після коми.

    Змінні контексту, передані в шаблон:
    - 'user': Увійшовший користувач.
    - 'my_recipes': Рецепти, створені користувачем.
    - 'favorite_recipes': Рецепти, позначені як улюблені користувачем.
    - 'want_to_cook_recipes': Рецепти, які користувач хоче приготувати.
    - 'shopping_list': Форматований словник, що містить список покупок, з назвами інгредієнтів як ключами
      і їхніми відповідними кількостями як значеннями.

    Аргументи:
        request (HttpRequest): Об'єкт запиту.

    Повертає:
        HttpResponse: Відображена сторінка профілю з даними контексту.
